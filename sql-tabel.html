TABEL

Di dalam sebuah database dapat memiliki satu atau lebih tabel. Setiap tabel memiliki nama dan di dalam sebuah tabel terdapat banyak record. Berikut adalah contoh tabel Mahasiswa:

Diatas adalah sebuah tabel mahasiswa yang memiliki 5 record yang terdiri dari data berdasarkan atribut tabel, seperti NIM, nama, dan angkatan.
Untuk membuat tabel dengan SQL, gunakan sintaks berikut:

CREATE TABLE nama_tabel (
Atribut1 tipedata1,
Atribut2 tipedata2,
Atribut3 tipedata3,
…
);

Sehingga untuk membuat tabel mahasiswa dengan atribut NIM, Nama, dan Angkatan, maka sintaks yang perlu digunakan adalah:

CREATE TABLE mahasiswa (
NIM VARCHAR(10),
Nama VARCHAR(25),
Angkatan INTEGER
);

Jika sintaks diatas dijalankan, maka akan terbentuk tabel kosong seperti berikut:

NOTE :

Ada banyak tipe data yang dapat digunakan di SQL, namun yang paling umum digunakan adalah Int, Real, Boolean, Char, Varchar (ini sama dengan string), Time (jam, menit, detik), Date (Tahun, Bulan, Hari) dan Timestamp (tahun, bulan, hari, jam, menit, detik). Untuk tipe data varchar juga ditulis berapa jumlah karakter maksimal nya. (Pada contoh terdapat definisi atribut “NIM varchar(10)” yang berarti NIM hanya dapat diisi oleh string maksimal 10 karakter)

Kita juga dapat menambahkan, menghapus, dan mengubah tipe data atribut pada sebuah tabel yang sudah dibuat. Untuk menambahkan atribut, gunakan sintaks berikut:


ALTER TABLE nama_tabel ADD nama_atribut tipe_data_atribut

Untuk menghapus atribut, gunakan:


ALTER TABLE nama_tabel DROP nama_atribut tipe_data_atribut

Sedangkan untuk mengubah tipe data atribut, sintaks yang digunakan bergantung pada aplikasi yang digunakan:
SQL Server / MS Access:


ALTER TABLE nama_tabel ALTER COLUMN nama_atribut tipe_data_atribut

My SQL / Oracle sebelum versi 10G:


ALTER TABLE nama_tabel MODIFY COLUMN nama_atribut tipe_data_atribut

Oracle 10G dan kedepannya:


ALTER TABLE nama_tabel MODIFY nama_atribut tipe_data_atribut

Sebagai contoh jika kita mempunyai tabel mahasiswa seperti diatas dan menjalankan sintaks berikut:

ALTER TABLE mahasiswa ADD IPK INT;
ALTER TABLE mahasiswa DROP angkatan;
ALTER TABLE mahasiswa MODIFY COLUMN Nama(50);

Maka tabel mahasiswa akan menjadi seperti ini:


Atribut IPK ditambahkan di tabel mahasiswa dan atribut angkatan dihapus. Lalu tipe data Nama diubah dari varchar(25) menjadi varchar(50).
Jika kita tidak memerlukan sebuah tabel lagi, maka kita dapat menghapus sebuah tabel tersebut dengan sintaks:


DROP TABLE nama_tabel

Berarti jika kita ingin menghapus tabel mahasiswa, maka kita dapat menulis:


DROP TABLE mahasiswa

PRIMARY DAN FOREIGN KEY

Di dalam sebuah tabel sangat mungkin terdapat dua atau lebih record.
Untuk membedakan satu record dengan yang lainnya, dibutuhkan nilai dalam record yang unik untuk mengidentifikasi setiap record.
Nilai ini adalah PRIMARY KEY, atribut tabel yang nilai di setiap record selalu unik agar setiap record dapat teridentifikasi
Sebuah tabel hanya boleh memiliki satu Primary Key.
Untuk membuat atribut1 menjadi primary key pada sebuah tabel, maka sintaks yang digunakan adalah:

CREATE nama_tabel (
Atribut1 Tipe_data1 PRIMARY KEY,
Atribut2 Tipe_data2,
Atribut2 Tipe_data2,
.....
AtributN Tipe_data_n
);

Dalam MySQL sintaks yang digunakan sedikit berbeda, yaitu:

CREATE nama_tabel (
Atribut1 Tipe_data,
Atribut2 Tipe_data2,
Atribut2 Tipe_data2,
.....
AtributN Tipe_data_n
PRIMARY KEY (Atribut1)
);

NOTE :

Pada sintaks diatas Atribut pertama menjadi primaty key, namun sebenarnya primary key tidak harus di atribut pertama. Primary key pada sebuah tabel bisa atribut apapun, asalkan hanya satu.

Jika kita ingin membuat tabel mahasiswa dengan atribut NIM, nama, dan angkatan dengan NIM adalah primary key, maka code yang ditulis adalah:

CREATE TABLE mahasiswa(
NIM VARCHAR(10) PRIMARY KEY,
Nama VARCHAR(30),
Angkatan YEAR
);

Berikut adalah tabel yang terbentuk:

Perhatikan tabel mahasiswa yang telah diisi dengan dua record di bawah ini

Dalam tabel mahasiswa boleh terdapat 2 record dengan nilai atribut Nama yang sama, karena atribut Nama bukan primary key sehingga Nama tidak perlu unik. Namun disaat kita ingin mengisi tabel dengan record baru NIM=1, Nama=”Ahmad”, Angkatan=”2018”, terjadi error karena sudah ada record dengan nilai NIM=1 dan NIM adalah primary key sehingga record barusan tidak bisa dimasukkan ke tabel.

Untuk membuat atribut pada tabel yang sudah ada menjadi primary key, gunakan sintaks:


ALTER TABLE nama_tabel ADD PRIMARY KEY (nama_atribut);

Untuk menghapus primary key pada tabel, gunakan sintaks:


ALTER TABLE nama_tabel DROP PRIMARY KEY;

Foreign key digunakan untuk menghubungkan dua tabel.
Foreign key adalah atribut tabel yang merujuk pada primary key tabel lain.
Tabel yang mengandung sebuah foreign key disebut tabel child, dan tabel yang mengandung primary key yang dijadikan foreign key disebut tabel parent.

Tabel Mahasiswa

Tabel Dosen

Pada tabel mahasiswa terdapat atribut FOREIGN KEY Dosen Wali yang merujuk pada PRIMARY KEY NIP di tabel Dosen. Untuk membuat tabel mahasiswa yang memiliki foreign key, maka code yang perlu ditulis adalah:

CREATE TABLE Mahasiswa (
NIM VARCHAR(4) PRIMARY KEY,
Nama VARCHAR(20),
Dosen_Wali VARCHAR(4) FOREIGN KEY REFERENCES Dosen(NIP)
);

Jika misalkan tabel mahasiswa dan dosen sudah ada namun atribut Dosen Wali pada mahasiswa masih belum menjadi foreign key, maka code ini dapat membuat atribut tersebut menjadi foreign key:


ALTER TABLE mahasiswa ADD FOREIGN KEY (Dosen_Wali) REFERENCES Dosen(NIP);

Untuk menghapus foreign key pada tabel mahasiswa, gunakan code:


ALTER TABLE mahasiswa DROP FOREIGN KEY Dosen_Wali;
